How to Use the Data Sanitizer

Quick Start

  • Paste text: Paste sensitive content into the Input box (auto-sanitizes on paste)
  • Upload file: Click the dropzone, use the file picker, or drag & drop a text file
  • Review changes: Check the diff preview and redaction summary
  • Export: Copy or download the sanitized output

What Gets Redacted

  • [PRIVATE_KEY] — SSH/TLS private keys
  • [JWT] — JSON Web Tokens
  • [GITLAB_PAT] — GitLab Personal Access Tokens
  • [AWS_ACCESS_KEY_ID] — AWS access keys
  • [KUBE_SECRET] — Kubernetes secret references
  • [EMAIL] — Email addresses
  • [IP] / [IP6] — IPv4 and IPv6 addresses
  • [FQDN] — Fully qualified domain names
  • [USER]@[HOST] — Shell prompts and usernames
  • [PATH] — File system paths (Windows & Unix)
  • [MAC] — MAC addresses
  • [UUID] — UUIDs
  • [CREDIT_CARD] — Valid credit card numbers (Luhn check)
  • [FINNISH_ID] — Finnish personal IDs
  • [IBAN] — Finnish IBANs
  • password=[REDACTED] — Password/token values

Features

  • Diff Preview: See exactly what changed (red = removed, green = added)
  • Show only changed lines: Toggle checkbox to hide unchanged lines
  • Redaction Summary: Count of each redaction type
  • Multiple Export Options: Copy cleaned text, copy diff, or download as file
  • File Size Limit: Maximum 2 MB per file

Use Cases

  • Preparing logs/configs for AI prompts (ChatGPT, Copilot, etc.)
  • Sharing debug output in tickets or public forums
  • Sanitizing error messages before posting to Stack Overflow
  • Redacting sensitive data from documentation

Tips

  • Always review the diff before sharing—automated tools can miss context-specific secrets
  • For shell output, keep command structure visible while hiding identifying info
  • Download sanitized files for large logs instead of copy/paste
